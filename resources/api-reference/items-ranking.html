Items: Ranking

Returns a ranking of items, including their usage stats.

HTTP Request

GET /api/items/ranking
GET /api-xml/items/ranking
Query parameters
Required parameters
format string
Name of the format you want to get the stats from.
Example: [Gen 7] Ubers
Optional parameters
baseline number
Ranking baseline you want to apply.
Only certain baselines are available.
If you do not specify a baseline. The default baseline is used.
Examples: 0, 1500, 1630, 1695, 1760, 1825
month string
The month when the usage stats were taken. Format: YYYY-MM
Example: 2018-10
By default, it uses the latest month.
pos string
Filters by the position in the ranking. List of conditions.
Format: condition1, condition2...
Types of conditions:
  • =number - Equals
  • !=number - Not equals
  • >number - Greater than
  • >=number - Greater than or equals
  • <number - Lesser than
  • <=number - Lesser than or equals

Example: <=50
usage string
Filters by the usage (%). List of conditions.
Format: condition1, condition2...
Types of conditions:
  • =number - Equals
  • !=number - Not equals
  • >number - Greater than
  • >=number - Greater than or equals
  • <number - Lesser than
  • <=number - Lesser than or equals

Example: <=50
raw string
Filters by the raw count. List of conditions.
Format: condition1, condition2...
Types of conditions:
  • =number - Equals
  • !=number - Not equals
  • >number - Greater than
  • >=number - Greater than or equals
  • <number - Lesser than
  • <=number - Lesser than or equals

Example: <=50
id string
Chooses a single item.
Example: leftovers
return-as string
Sets the format for the result.
Set as xml to get the result as XML.
Set as json to get the result as JSON.
Default format is json.

Response

If you requested the response as JSON, the method will return a body with the following structure:

{
... "result": {
...... "feature": "items/ranking",
...... "month": {
......... "year": number,
......... "month": number
...... },
...... "format": {
......... "id": string,
......... "name": string
...... },
...... "total": number,
...... "items": [
......... {
............ "id": string,
............ "name": string,
............ "pos": number,
............ "usage": number,
............ "raw": number
......... }, ...
...... ]
... }
}

If you requested the response as XML, the method will return a body with the following structure:

<result>
...<feature> items/ranking</feature>
...<year> number</year>
...<month> number</month>
...<format id= string> Format name (string)</format>
...<baseline> number</baseline>
...<total> number</total>
...<ranking>
......<item id= string pos= number usage= number raw= number> Item name (string)</item>
...... ...
...</ranking>
</result>
Properties
feature string
API feature you requested.
year number
Year when the usage stats were taken.
month number
Month of the year (01 - 12) when the usage stats were taken.
format string
Format name.
baseline number
Baseline used for the usage stats compilation.
total number
Total items counted.
id string
Item identifier.
name string
Item name.
pos number
Position of the item in the ranking.
usage number
Usage of the item.
raw number
Raw count of the item.

Errors

Status Error Code Description
404 MONTH_NOT_AVAILABLE This error means the month you specified was not found.
Maybe you spell it wrong or the month is not available.
404 FORMAT_NOT_FOUND This error means the format you specified was not found.
Maybe you spell it wrong or the format is not available.
404 BASELINE_NOT_AVAILABLE This error means you specified a baseline that is not available.
