Metagame: Formats list

Returns the list of available formats for the "Metagame" feature.

HTTP Request

GET /api/metagame/formats
GET /api-xml/metagame/formats
Query parameters
Optional parameters
month string
The month when the usage stats were taken. Format: YYYY-MM
Example: 2018-10
By default, it uses the latest month.
baseline string
Filters by the baseline. List of conditions.
Format: condition1, condition2...
Types of conditions:
  • =number - Equals
  • !=number - Not equals
  • >number - Greater than
  • >=number - Greater than or equals
  • <number - Lesser than
  • <=number - Lesser than or equals

Example: >=1500, <2000
stalliness string
Filters by the stalliness indicator. List of conditions.
Format: condition1, condition2...
Types of conditions:
  • =number - Equals
  • !=number - Not equals
  • >number - Greater than
  • >=number - Greater than or equals
  • <number - Lesser than
  • <=number - Lesser than or equals

Example: >=0.5
top-style string
Filters by the most used playstyle.
search-for string
The name or part of the name of the format you are looking for.
return-as string
Sets the format for the result.
Set as xml to get the result as XML.
Set as json to get the result as JSON.
Default format is json.

Response

If you requested the response as JSON, the method will return a body with the following structure:

{
... "result": {
...... "feature": "metagame/formats",
...... "month": {
......... "year": number,
......... "month": number
...... },
...... "formats": [
......... {
............ "id": string,
............ "name": string,
............ "baseline": number,
............ "stalliness": number,
............ "topStyle": string
......... }, ...
...... ]
... }
}

If you requested the response as XML, the method will return a body with the following structure:

<result>
...<feature> metagame/formats</feature>
...<year> number</year>
...<month> number</month>
...<formats>
......<format id= string baseline= number stalliness= number topStyle= string> Format name (string)</format>
...... ...
...</formats>
</result>
Properties
feature string
API feature you requested.
year number
Year when the usage stats were taken.
month number
Month of the year (01 - 12) when the usage stats were taken.
id string
Format identifier.
name string
Format name.
baseline number
Baseline used for the usage stats compilation.
stalliness number
Stalliness indicator.
More negative = more offensive. More positive = more stall
topStyle string
The most used playstyle for the format.

Errors

Status Error Code Description
404 MONTH_NOT_AVAILABLE This error means the month you specified was not found.
Maybe you spell it wrong or the month is not available.
